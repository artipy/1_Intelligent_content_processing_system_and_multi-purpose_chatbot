### Название проекта: **"Интеллектуальная система обработки контента и многоцелевой чат-бот"**

**Краткое описание:**
Цель проекта — создать универсальное Python-приложение, которое умеет не просто отвечать на вопросы, а выполнять сложные задачи по редактированию, суммаризации и классификации данных с использованием API. Мы пройдем путь от простого запроса до создания полноценного чат-бота с памятью и заданным характером поведения.

**Используемые технологии:**
*   **Язык программирования:** Python (базовые знания списков, словарей и циклов).
*   **Библиотека:** OpenAI Python Library для взаимодействия с API.
*   **Инструменты:** f-strings для формирования динамических промптов, triple quotes для многострочного текста.
*   **Модели:** Любые LLM, совместимые с OpenAI API (многие провайдеры предоставляют бесплатный пробный период или бесплатные токены, как упоминается в источниках в контексте OpenAI).

---

### Пошаговое выполнение проекта

1.  **Этап: Установка соединения и первый запрос**
    *   **Описание:** Настройка клиента OpenAI, аутентификация с помощью API-ключа и отправка первого запроса к эндпоинту `chat completions`.
    *   **Положительный результат:** Код успешно подключается к API и выводит текстовое содержимое ответа, извлеченное из объекта `chat completion` через атрибуты `choices.message.content`.

2.  **Этап: Инструмент автоматической суммаризации и редактирования**
    *   **Описание (Фича):** Добавление функционала для сжатия длинных текстов и изменения тона или данных в тексте (например, замена имен или должностей в биографии) с использованием `f-strings` для вставки переменных в промпт.
    *   **Положительный результат:** Модель возвращает краткую выжимку длинного текста или корректно отредактированную версию исходного текста.

3.  **Этап: Управление параметрами генерации (Randomness & Length)**
    *   **Описание (Фича):** Внедрение параметров `temperature` для контроля случайности (от 0 для точности до 2 для креативности) и `max_completion_tokens` для ограничения длины ответа и контроля затрат.
    *   **Положительный результат:** При `temperature=0` ответы становятся стабильными и предсказуемыми, а при задании `max_completion_tokens` ответ строго обрезается или завершается в рамках лимита.

4.  **Этап: Классификатор данных с использованием Shot Prompting**
    *   **Описание (Фича):** Реализация классификации текста (например, анализ тональности отзывов от 1 до 5) с использованием техник `one-shot` или `few-shot prompting` (предоставление примеров в промпте).
    *   **Положительный результат:** Модель выдает ответ в строго заданном формате (например, только цифру или категорию), следуя логике предоставленных примеров, без лишних пояснений.

5.  **Этап: Формирование личности ассистента и "Guardrails"**
    *   **Описание (Фича):** Использование системной роли (`system role`) для задания поведения (например, "ты лаконичный репетитор по Python") и установка ограничений (guardrails), чтобы модель отказывалась выполнять нежелательные действия (например, давать финансовые советы).
    *   **Положительный результат:** Модель придерживается заданного стиля общения и выдает заранее подготовленный отказ при попытке нарушить установленные правила.

6.  **Этап: Структурированные примеры через роль Assistant**
    *   **Описание (Фича):** Улучшение точности ответов путем добавления в список сообщений (`messages`) пар "user-assistant", которые служат идеальными примерами диалога.
    *   **Положительный результат:** Модель имитирует стиль и формат ответов, заданный в цепочке сообщений роли `assistant`.

7.  **Этап: Полноценный чат-бот с историей сообщений (Multi-turn conversation)**
    *   **Описание (Фича):** Создание цикла, который принимает ввод пользователя, сохраняет ответ модели и добавляет всю историю переписки обратно в список `messages` для следующего запроса.
    *   **Положительный результат:** Бот "помнит" контекст предыдущих фраз (например, знает имя пользователя, названное в начале беседы) и может отвечать на уточняющие вопросы.

### Описание итогового результата проекта
Проект считается успешно выполненным, если создано Python-приложение, представляющее собой **интеллектуального чат-бота**, который:
1.  Обладает заданной "личность" и соблюдает правила безопасности (через `system message`).
2.  Способен выполнять специфические задачи обработки текста (суммаризация, классификация) с высокой точностью благодаря примерам в промпте (`shot prompting`).
3.  Поддерживает непрерывный диалог, сохраняя контекст беседы в памяти.
4.  Позволяет разработчику гибко настраивать креативность и длину ответов через параметры API.
